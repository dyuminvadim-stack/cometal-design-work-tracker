# Process Tracker Kit

Локальная болванка для упаковки рабочего плана в визуальный трекер.

## Когда использовать

Фраза-триггер:

> хочу этот план упаковать в трекер

После этого я задаю короткий набор вопросов, превращаю ответы в JSON-план и загружаю его в локальное приложение.

## Вопросы для упаковки плана

1. Как называется трекер?
2. Какой результат должен быть получен в конце процесса?
3. Какие этапы есть в плане и в каком порядке они идут?
4. Какие задачи или артефакты входят в каждый этап?
5. Что считается готовым результатом для каждой задачи?
6. Есть ли сроки, приоритеты, ответственные или зависимости?
7. Нужно ли хранить итоговые файлы рядом с задачами?

## Формат JSON

Приложение принимает файл `.json` в таком виде:

```json
{
  "eyebrow": "Design plan",
  "name": "Design Work Tracker",
  "intro": "Короткое описание процесса и ожидаемого результата.",
  "exportName": "design-work-tracker-export",
  "taskGroupLabel": "TASKS",
  "stages": [
    {
      "id": 1,
      "title": "Этап 1. Название этапа",
      "description": "Что происходит на этом этапе.",
      "status": "not_started",
      "prompts": [
        {
          "title": "artifact_name.md",
          "description": "Что нужно сделать",
          "targetFile": "artifact_name.md",
          "sourceFile": "Исходные материалы",
          "promptText": "Инструкция или задача для выполнения.",
          "completed": false
        }
      ]
    }
  ]
}
```

## Минимальный формат

Если план ещё сырой, можно дать только этапы и задачи. Остальное приложение заполнит дефолтами.

```json
{
  "name": "Название трекера",
  "stages": [
    {
      "title": "Этап",
      "tasks": [
        {
          "title": "Задача",
          "description": "Описание задачи"
        }
      ]
    }
  ]
}
```

## Как работать

1. Запустить приложение: `npm run dev`.
2. Нажать `Import`.
3. Выбрать JSON-план.
4. Открывать задачи, копировать инструкции, прикреплять результаты.
5. Нажать `Export`, чтобы получить ZIP с состоянием процесса и файлами результата.

## Что хранится локально

Активный трекер, прогресс и прикреплённые файлы сохраняются в `localStorage` браузера. Кнопка `Reset` сбрасывает прогресс текущего трекера. Кнопка `Default` возвращает исходный COMETAL-трекер.
