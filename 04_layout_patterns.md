# Layout Patterns для PRODUCT SCREENS

Источник: https://www.figma.com/design/wt7rhRt3u2xq4t2ajUZ90R/COMETAL--2.0-18.06?m=auto&t=L9FxkjNzn3D3mWHi-6

Область анализа: только страницы из категории `PRODUCT SCREENS`.

Не анализировались: компоненты, дизайн-система, визуальный стиль, страницы `BUSINESS FLOWS`, `EXPERIMENTS`, `ARCHIVE`.

Основание: структурные маркеры из Figma MCP: названия страниц, секций и фреймов. Классификация сделана по заданным категориям: `Dashboard`, `List`, `Kanban`, `Entity Card`, `Wizard`, `Form`, `Table`, `Modal`, `Settings`, `Timeline`.

| Экран | Основной тип | Дополнительные типы | Основание |

## Повторяющиеся шаблоны

### 1. Operational List / Table Workspace

**Структура**

- Глобальная навигация / меню.
- Заголовок рабочей области.
- Инструменты над списком: фильтры, поиск, действия, экспорт/импорт.
- Основная зона: таблица, список строк или rows.
- Опционально: боковой фильтр, audit log, popup для подтверждения действия.

**Пример экранов**

- Импорт платежей
- Экспорт план-факта платежей
- Страница "Отгрузки"
- Задачи
- Металлотрейдинг

**Количество использований:** 5

### 2. Entity Card / Entity Workspace

**Структура**

- Шапка сущности: название, статус, основные действия.
- Основная информация по сущности.
- Блоки/виджеты: связанные заявки, документы, оплаты, отгрузки, задачи, связи.
- Вкладки или секции для подробных данных.
- Опционально: таблицы, графики, коммуникации, дополнительная панель.

**Пример экранов**

- Карточка компании
- КАРТОЧКА КОМПАНИИ
- Металлотрейдинг
- Карточка отгрузки - модальное окно

**Количество использований:** 4

### 3. Modal / Popup Workflow

**Структура**

- Затемнение или контекстный overlay.
- `Popup-Content`.
- `Popup-Header`.
- Основной `Content`.
- Опционально: `Popup-Footer`, кнопки действий, подтверждение, форма или таблица внутри модального окна.

**Пример экранов**

- Карточка отгрузки - модальное окно
- Задачи
- Импорт платежей
- Экспорт план-факта платежей
- Отвественные по заявке
- Канбан оплат

**Количество использований:** 6

### 4. Kanban Workflow Board

**Структура**

- Колонки статусов или этапов.
- Заголовки колонок.
- Карточки задач/объектов внутри колонок.
- Сценарии переходов между статусами.
- Опционально: модальные действия, wizard, график/таймлайн платежей.

**Пример экранов**

- Канбан оплат

**Количество использований:** 1

### 5. Form / Wizard Flow

**Структура**

- Контекст процесса или сущности.
- Последовательность шагов или форма ввода.
- Валидация/алерты.
- Действия подтверждения, сохранения или перехода дальше.
- Опционально: `Stepper`, выбор ответственного, загрузка файла.

**Пример экранов**

- Канбан оплат
- Импорт платежей
- Металлотрейдинг
- Страница "Отгрузки"
- Задачи

**Количество использований:** 5

### 6. Settings / Assignment Panel

**Структура**

- Контекст сущности: заявка или задача.
- Список ролей/ответственных.
- Поиск или выбор пользователя.
- Действие назначения/изменения ответственного.
- Опционально: модальное окно подтверждения.

**Пример экранов**

- Отвественные по заявке
- Канбан оплат
- Задачи

**Количество использований:** 3

### 7. Dashboard / Monitoring Workspace

**Структура**

- Обзор сущности или раздела.
- Виджеты мониторинга.
- Сводные блоки.
- Таблицы/графики/связи.
- Переходы к связанным спискам и карточкам.

**Пример экранов**

- Карточка компании
- КАРТОЧКА КОМПАНИИ
- Страница "Отгрузки"

**Количество использований:** 3

### 8. Timeline / Audit Log

**Структура**

- Хронологический или журналируемый список событий.
- Событие / действие / статус.
- Автор или источник изменения.
- Время или порядок события.
- Опционально: фильтр и детализация события.

**Пример экранов**

- Импорт платежей
- Канбан оплат
- Карточка компании

**Количество использований:** 3

## Уникальные layout patterns

1. **Navigation Shell** — общий каркас продукта: меню, рабочая область, tools/additional панель. Нужен почти всем продуктовым экранам как базовая оболочка.
2. **Operational List / Table Workspace** — основной шаблон для платежей, задач, отгрузок и экспортно-импортных сценариев.
3. **Entity Card / Entity Workspace** — шаблон карточки сущности: компания, заявка/направление, отгрузка.
4. **Modal / Popup Workflow** — единый шаблон модальных действий, подтверждений, просмотра и создания.
5. **Form / Wizard Flow** — шаблон пошагового или формового процесса: загрузка, расчет, создание, назначение.
6. **Kanban Workflow Board** — отдельный шаблон статусов/колонок для оплат.
7. **Settings / Assignment Panel** — шаблон назначения ответственных и настройки участников процесса.
8. **Dashboard / Monitoring Workspace** — шаблон обзора сущности или раздела через виджеты, графики, связи и сводки.
9. **Timeline / Audit Log** — шаблон истории событий, аудита и графиков/журналов.

## Итог

Для всех `PRODUCT SCREENS` в текущем файле достаточно **9 уникальных layout patterns**.

Если считать только полноценные экранные шаблоны без вспомогательных overlay-паттернов, ядро системы сводится к **5 базовым шаблонам**:

1. Operational List / Table Workspace
2. Entity Card / Entity Workspace
3. Form / Wizard Flow
4. Kanban Workflow Board
5. Dashboard / Monitoring Workspace

Остальные 4 паттерна являются поддерживающими:

1. Navigation Shell
2. Modal / Popup Workflow
3. Settings / Assignment Panel
4. Timeline / Audit Log
