Цель проверки: оценить полноту файла `COMETAL (Основной)` через Figma MCP без построения доменной архитектуры и без карты продукта.

## Методика

Через Figma MCP были выполнены:

- чтение списка всех Figma pages;
- подсчет объектов, секций и фреймов;
- проверка node из ссылки `9702:917`;
- классификация страниц по структурным признакам и названиям.

Важно: node `9702:917` из ссылки оказался не продуктовым экраном, а Figma page `🟩 Outlined Input` в компонентном разделе. Это объясняет, почему простой анализ первой страницы ранее видел только `ОБЛОЖКА`.

## Вывод

Файл **не является почти пустым**.

Он содержит:

- 142 Figma pages;
- 1474 объекта;
- 60 секций;
- 60 фреймов;
- выраженную структуру дизайн-системы и компонентных страниц;
- большое количество страниц с названиями продуктовых зон: заявки, компании, канбаны, звонки, документы, отгрузки, оплаты, авторизация, создание заявки, коммуникации.

Но важное уточнение: по точным счетчикам MCP основная масса объектов находится не в продуктовых экранах, а в компонентной странице `🟩 Outlined Input`. Большинство продуктовых страниц в полученном exact-count диапазоне либо пустые, либо содержат только секции без фреймов экранов.

Итоговая оценка:

**файл содержит значительный объём продуктовой логики, который не был учтён в предыдущем анализе.**

При этом для полноценной инвентаризации экранов нужен повторный MCP-проход после снятия rate limit, особенно по страницам 101-142, где находятся продуктовые зоны `Авторизация`, `Создание заявки`, `Заявка`, `Компания`, `ОПЛАТЫ`, `Список оплат`, `Карточка оплаты` и связанные разделы.
